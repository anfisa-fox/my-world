# Назначение документа

Данный документ фиксирует результаты Sprint 002.

Документ является неизменяемым историческим артефактом проекта и отражает состояние системы на момент завершения Sprint.

Все последующие изменения оформляются отдельными Sprint Handoff.

---

# Цель Sprint

Расширить Creator Studio поддержкой полноценного Author Mode для раздела Gallery.

Основная задача заключалась в том, чтобы автор мог самостоятельно публиковать новые иллюстрации без знания Git, GitHub, Markdown или программирования.

---

# Реализовано

## Gallery Author Mode

Полностью реализован пользовательский сценарий публикации новой иллюстрации.

Поддерживаются:

* Author Mode;
* форма публикации;
* загрузка изображения;
* описание;
* теги;
* период;
* автоматическая генерация slug;
* автоматическое создание Markdown;
* публикация изображения;
* GitHub Commit;
* автоматический Cloudflare Deploy.

---

## Image Upload

Добавлена загрузка изображений непосредственно из интерфейса Gallery.

Изображение автоматически сохраняется в:

```text
public/images/artworks/
```

Автор не взаимодействует с файловой структурой проекта.

---

## Markdown Generation

При публикации автоматически создаётся Markdown-файл.

Заполняются:

* title
* slug
* image
* description
* tags
* period
* featured
* createdAt

---

## Timestamp Strategy

Принято новое правило хранения времени создания контента.

Все новые записи используют полный ISO timestamp.

Пример:

```yaml
createdAt: "2026-06-29T17:42:18.531Z"
```

Старые записи продолжают поддерживаться.

Gallery сортирует публикации через Date(), обеспечивая совместимость обоих форматов.

---

## Gallery Sorting

Gallery больше не зависит от порядка файлов в файловой системе.

Сортировка выполняется по времени создания.

Это правило становится эталонным для остальных разделов проекта.

---

## Pending Publications

После публикации автор сразу возвращается в Gallery.

До завершения Cloudflare Deploy отображается placeholder.

Поддерживается:

* несколько одновременных публикаций;
* sessionStorage;
* TTL очистка;
* functional state updates;
* сохранение состояния между переходами внутри сайта.

---

# Архитектурные решения

Во время Sprint приняты следующие решения.

## Author Mode

Единственным источником состояния Author Mode является:

```
AuthorModeProvider
```

Компоненты не работают напрямую с localStorage.

---

## Pending Publications

Pending-публикации являются локальным UX-состоянием Gallery.

Они не относятся к глобальному состоянию приложения.

Используется sessionStorage.

---

## Content Repository

Все операции публикации продолжают использовать общий слой:

```text
functions/lib/content.ts
```

Gallery не содержит собственной логики работы с GitHub.

---

## Production Pipeline

Процесс публикации полностью автоматизирован.

```text
Author

↓

Gallery Author Mode

↓

Cloudflare Function

↓

GitHub Repository

↓

Cloudflare Pages Deploy

↓

Production Gallery
```

# Known Limitations MVP

Зафиксированы следующие ограничения.

* уже открытая вкладка Gallery не получает автоматически новый статический контент после завершения Cloudflare Deploy;
* placeholder может оставаться до повторной загрузки страницы;
* это ограничение связано со Static Export и не считается дефектом механизма публикации.

Для MVP дополнительных механизмов синхронизации не требуется.
