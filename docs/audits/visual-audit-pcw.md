# Visual Design Audit — Personal Creative World
**Дата:** июнь 2026  
**Версия сайта:** localhost:3010 (pre-demo)  
**Аудитор:** арт-директор проекта

---

## Общая оценка

**54% соответствия дизайн-концепции**

Скелет реализован: палитра, шрифты, структура Home. Философия «окон в мир» на главной страницы узнаётся. Однако магия «скетчбука, ожившего в браузере» пока не передана. Пять правок из P0 переведут проект на ~80% за один день работы.

---

## 1. Философия — Digital Sanctuary, Magic, Sketchbook

**🟡 Частично соответствует**

### ✓ Что работает
Структура Home («Preview of the World») реализована верно. Три фрагмента — галерея / персонаж / стена — с принципом обрыва и лейблами соответствуют концепции.

### ✗ Проблемы

**Magic и Sanctuary не ощущаются.**  
Интерфейс тихий, но не магический. Нет ни одного момента «вау». Плейсхолдер Moon Girl на главной (лунный символ на бежевом) слишком нейтрален как центральный образ.  
→ После загрузки реальных иллюстраций это изменится само. До тех пор — добавить на hero более насыщенный placeholder (#D4E0D0) или тонкий паттерн из символов ✦.

**Sketchbook feeling отсутствует полностью.**  
Нет ни одного элемента «ручной работы». Все радиусы, отступы, типографика — чистый цифровой интерфейс без личности.  
→ Добавить Pinned Corner (кружок #E8C4A8, 8px) на все карточки галереи. Это минимальная деталь с максимальным эффектом.

## 2. Визуальная композиция

**🟡 Частично соответствует**

### ✓ Что работает
Home: ритм и вертикальные интервалы между секциями правильные. Иерархия читается.

### ✗ Проблемы

**Gallery: горизонтальный формат карточек разрушает искусство.**  
Все иллюстрации обрезаются в горизонтальные полосы ~180px высотой. «Cloud Harbor» и «Dream Library» — сложные детализированные панорамы — теряют 70% содержания. Это критичная проблема: иллюстрации — главный герой сайта, а интерфейс их скрывает.  
→ Сетка 3 колонки, aspect-ratio 3/4 (вертикальные карточки). Именно так задано в дизайн-системе.

**Characters: пустой экран с половиной страницы воздуха.**  
Три горизонтальные строки + полстраницы пустого пространства. Ощущение незавершённости, а не лаконичности.  
→ Сетка 2 колонки с карточками. Добавить контент (4–6 персонажей).

## 3. Типографика

**🔴 Требует переработки**

### ✓ Что работает
Cormorant Garamond подключён. «Загляни в мой мир» на главной читается правильно — шрифт живой и соответствует концепции.

### ✗ Проблемы

**«Gallery», «Characters», «Wall» — h1 жирный, не Cormorant.**  
Заголовки страниц — жирный гротеск. Создаёт диссонанс с нежной навигацией и нарушает визуальный язык.  
→ `font-family: 'Cormorant Garamond', serif; font-weight: 300; font-size: 36px` на всех page-level заголовках.

**Страница работы: заголовок «Cloud Harbor» — bold без Cormorant.**  
Самый важный текст на странице работы выглядит как заголовок новостного сайта.  
→ Cormorant Garamond 300 для всех title на страницах `/gallery/[slug]` и `/wall/[slug]`.

**Дата — сырая ISO-строка на трёх экранах.**  
`Wed Jun 24 2026 03:00:00 GMT+0300 (Москва, стандартное время)` — техническая строка видна пользователю на страницах работы, персонажа и стены. Убивает атмосферу полностью.  
→ Утилита `formatDate()`:
```ts
new Date(str).toLocaleDateString('ru-RU', {
  day: 'numeric', month: 'long', year: 'numeric'
})
// Результат: «24 июня 2026»
```
Применить везде. P0.

## 4. Цветовая система

**🟢 Соответствует**

Палитра реализована точно: фон #FAFAF8, sage-акцент, peach — всё на месте. Интерфейс не выглядит «бледным» — он спокойный, что и было задачей.

### Единственная проблема
Плейсхолдер Moon Girl на Home (светло-зелёный прямоугольник с символом) слишком нейтрален. После добавления реальной иллюстрации исчезнет само.  
→ Временно: placeholder background #D4E0D0 вместо текущего.

## 5. Карточки контента — Gallery, Characters, Wall

**🔴 Требует переработки**

**Gallery: карточки = каталог, не «окна в мир».**  
Горизонтальный формат + border со всех сторон + заголовок bold под изображением = стандартный CMS-шаблон. Нет Pinned Corner, нет атмосферы.  
→ 3 колонки, aspect 3/4, border-radius 4px, Pinned Corner, DM Mono для метаданных.

**Characters: горизонтальный список без образа.**  
Маленькая сломанная иконка слева + имя + текст = строка в таблице. Никакого ощущения «познакомиться с персонажем».  
→ Круглый аватар 52px с инициалом, имя в Cormorant, описание с градиентом-обрывом. Luna Windfeather с реальной иллюстрацией уже выглядит хорошо — применить этот подход к структуре всех карточек.

**Wall: карточки закрытые, не приглашают.**  
Border со всех сторон делает каждую запись «завершённой коробкой». Принцип «неполноты как приглашения» не работает.  
→ Убрать border, добавить `border-left: 2.5px solid #8BA888`. Текст обрывать градиентом. Ссылка «читать дальше →» есть на Home — нужна и на странице Wall.

## 6. Главная страница — Windows Into the World

**🟡 Частично соответствует**

### ✓ Что работает
Структура «Preview of the World» реализована. Три фрагмента с лейблами («из галереи», «персонаж», «со стены»), обрывом текста и ссылками — концепция соблюдена.

### ✗ Проблемы

**Логотип в синей рамке — артефакт разработки.**  
«мой мир ✦» обведён focus outline браузера. Выглядит как баг на первом экране.  
→ `outline: none` на элементе логотипа.

**Правая колонка приветствия слабая.**  
Текст («Здесь живут рисунки, персонажи и мысли...») мелкий и выровнен вправо — читается труднее.  
→ `font-size: 13px`, `color: #8A8780`, `text-align: right` оставить.

## 7. Header

**🟡 Частично соответствует**

### ✓ Что работает
Cormorant Garamond, символ ✦, мелкие ссылки справа — всё правильно. Активная ссылка с underline работает.

### ✗ Проблемы

**Активная ссылка на Wall: рамка вместо underline.**  
Пункт «стена» обведён border-box браузера (синяя рамка). Это focus state, а не дизайн.  
→ `outline: none` + стиль активного пункта: только `border-bottom: 1.5px solid #8BA888`.

## 8. Общая атмосфера

**🟡 Частично соответствует**

Честный ответ: это уже не «пустой Next.js», но ещё не «тот самый мир».

**Хорошее:** палитра, навигация, структура Home, иллюстрации (когда они есть — Luna Windfeather прекрасна, атмосфера появляется сразу).

**Плохое:** Gallery убивает иллюстрации форматом, сырые даты на трёх экранах, отсутствие Pinned Corner, жирные page-заголовки.

Когда иллюстрации работают — магия появляется сама. Контент у проекта сильный. Интерфейс должен перестать ему мешать.

## Справка: дизайн-токены (источник истины)

```css
:root {
  --color-bg:          #FAFAF8;
  --color-surface:     #F0EDE8;
  --color-placeholder: #EDE9E2;
  --color-sage:        #8BA888;
  --color-peach:       #E8C4A8;
  --color-ink:         #2C2A26;
  --color-muted:       #A09D98;
  --color-border:      #E8E4DE;
}
```

**Шрифты:** Cormorant Garamond (300, 400, italic) — заголовки. DM Sans (300, 400, 500) — тело. DM Mono (300, 400) — даты, теги, метаданные.

**Pinned Corner:** `position: absolute; top: 6px; right: 6px; width: 8px; height: 8px; background: #E8C4A8; border-radius: 50%; box-shadow: 0 0 0 1.5px #FAFAF8;`

**Галерея:** `grid-template-columns: repeat(3, 1fr); gap: 12px;` — карточки `aspect-ratio: 3/4`.

**Wall:** `border-left: 2.5px solid #8BA888; border-radius: 0; padding: 14px 16px;`
