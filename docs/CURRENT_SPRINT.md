# Назначение документа

Данный документ описывает исключительно текущий Sprint.

Его задача — дать разработчику точное понимание текущей цели, утверждённого Scope и ближайших шагов.

Документ не содержит долгосрочных архитектурных решений или истории проекта.

После завершения Sprint документ полностью обновляется для следующего этапа разработки.

---

# Цель Sprint

Основная цель Creator Studio Sprint 2 — подтвердить продуктовую гипотезу полноценного Author Mode.

Автор должен получить возможность управлять всеми основными типами контента непосредственно из опубликованного сайта, не используя Git, GitHub или локальную разработку.

---

# Основная гипотеза

Проект проверяет следующую гипотезу:

> Если управление контентом становится естественной частью самого творческого мира, автору будет хотеться регулярно возвращаться на сайт и развивать его.

Именно эта гипотеза является главным критерием успешности Sprint.

---

# Что уже завершено

К началу текущего этапа реализовано:

* Production Deployment;
* Cloudflare Pages Infrastructure;
* GitHub Publishing Pipeline;
* Creator Studio Sprint 1;
* создание публикаций Wall;
* автоматическая генерация Markdown;

---

# Текущий Scope

В рамках данного Sprint допускается реализация только следующих задач.

## Gallery

* создание новых работ;
* Author Mode для Gallery;
* использование существующего Content Repository Layer.

---

## Characters

* создание новых персонажей;
* Author Mode для Characters;
* использование существующего Content Repository Layer.

---

## Backend

Допускается развитие существующего backend только в пределах уже принятой архитектуры Cloudflare Pages Functions.

Изменение архитектуры проекта в рамках Sprint не допускается.

---

# Out of Scope

В данный Sprint сознательно не входят:

* универсальная CMS;
* универсальный редактор;
* динамические формы;
* загрузка изображений;
* предпросмотр;
* черновики;
* удаление и редактирование Gallery и Characters (если не потребуются для завершения проверки гипотезы);
* рефакторинг без практической необходимости.

Все подобные идеи фиксируются отдельно и рассматриваются только после завершения пользовательской проверки.

---

# Критерий успешного завершения Sprint

Sprint считается успешно завершённым, если в production работают следующие сценарии:

* создание новой публикации Wall;
* создание новой работы Gallery;
* создание нового персонажа Characters;

при этом весь цикл выполняется через опубликованный сайт:

```text
Author Mode
        │
        ▼
Cloudflare Function
        │
        ▼
GitHub Repository
        │
        ▼
Automatic Cloudflare Deployment
        │
        ▼
Updated Production Website
```

Следующим шагом разработки является завершение Author Mode для Gallery.

После успешной проверки полного production-сценария аналогичный подход применяется для Characters.

---

# Правила Sprint

Во время реализации запрещается:

* самостоятельно расширять Scope;
* изменять утверждённую архитектуру;
* выполнять преждевременный рефакторинг;
* проектировать функциональность следующих Sprint.

Каждое изменение должно:

1. соответствовать утверждённому Scope;
2. решать одну логическую задачу;
3. сопровождаться проверкой;
4. завершаться отдельным commit.

Главный принцип текущего Sprint:

> **Сначала подтвердить продуктовую гипотезу. Только потом расширять возможности системы.**
