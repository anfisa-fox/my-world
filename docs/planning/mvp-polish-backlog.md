# Назначение документа

Данный документ сопровождает развитие продукта после первой публикации MVP.

В отличие от Release Notes, которые фиксируют уже выпущенные изменения, этот журнал хранит:

* результаты Smoke Test;
* наблюдения во время демонстраций;
* обратную связь автора;
* замечания внешних аудитов;
* идеи по развитию UX;
* продуктовые решения, принятые в процессе эксплуатации.

Документ является единым источником истины для всех утверждённых улучшений продукта.

---

# Принципы

Каждое предлагаемое изменение должно отвечать хотя бы одному из следующих критериев:

* улучшает удобство использования;
* усиливает эмоциональное восприятие мира;
* делает исследование сайта более естественным;
* упрощает навигацию;
* сохраняет простоту архитектуры.

Мы избегаем функциональности ради функциональности.

Первоначальная философия проекта всегда имеет приоритет над расширением возможностей.

# Утверждённые улучшения

## PCW-001 — Навигация между работами

**Статус:** Approved

**Источник:** Smoke Test · 2026-06-25

**Приоритет:** 🔴 High

### Решение

Добавить навигацию:

```text
← Предыдущая                     Следующая →
```

Размещать:

* сразу под заголовком;
* непосредственно над главным изображением.

### Причина

После просмотра изображения пользователь должен иметь возможность сразу перейти к следующей работе, не возвращаясь в галерею.

---

## PCW-002 — Кликабельные теги

**Статус:** Approved

**Источник:** Smoke Test · 2026-06-25

**Приоритет:** 🔴 High

### Решение

Сделать все теги кликабельными.

Предпочтительное направление:

* фильтрация внутри галереи;
* без создания отдельной сущности.

### Причина

В текущем дизайне теги визуально воспринимаются как интерактивные элементы.

Некликабельные теги создают ощущение незавершённого интерфейса.

---

## PCW-003 — Блок рекомендаций для работ

**Статус:** Approved

**Источник:** Smoke Test · 2026-06-25

**Приоритет:** 🟡 Medium

### Решение

После окончания истории добавить блок:

> **Другие уголки этого мира**

Для первой версии использовать работы с `featured: true`.

Исключать текущую работу.

### Причина

Страница должна приглашать пользователя продолжить исследование мира, а не заканчиваться сразу после текста.

---

## PCW-004 — Возврат в галерею

**Статус:** Approved

**Источник:** Smoke Test · 2026-06-25

**Приоритет:** 🟡 Medium

### Решение

После блока рекомендаций добавить ссылку:

```text
← Вернуться в галерею
```

### Причина

Это создаёт понятную точку завершения пользовательского сценария.

---

## PCW-005 — Унификация навигации страниц третьего уровня

**Статус:** Approved

**Источник:** Smoke Test · 2026-06-25

**Приоритет:** 🔴 High

### Решение

Использовать одинаковую навигацию:

```text
← Предыдущая                     Следующая →
```

на всех страницах третьего уровня:

* Gallery;
* Characters;
* будущих разделах при необходимости.

### Причина

Пользователь должен изучать мир по единым правилам навигации независимо от раздела.

---

## PCW-006 — «Другие уголки этого мира» для персонажей

**Статус:** Approved

**Источник:** Smoke Test · 2026-06-25

**Приоритет:** 🟡 Medium

### Решение

Заменить технический блок Gallery единым блоком рекомендаций:

> **Другие уголки этого мира**

В блок могут входить:

* другие персонажи;
* связанные работы;
* в будущем — записи со Стены.

### Причина

Пользователь должен путешествовать по миру, а не по файловой структуре сайта.

---

## PCW-007 — Осмысленные связи между персонажами и работами

**Статус:** Approved

**Источник:** Smoke Test · 2026-06-25

**Приоритет:** 🟡 Medium

### Решение

Связанные работы должны отображаться только при наличии явной смысловой связи.

Связи должны определяться контентной моделью проекта, а не структурой файлов.

### Причина

Случайные изображения создают ощущение незавершённой функциональности.

---

## PCW-008 — Исправление галереи персонажей

**Статус:** Approved

**Источник:** Smoke Test · 2026-06-25

**Приоритет:** 🔴 High

### Решение

Исправить отображение галереи персонажей.

Не допускать:

* битых изображений;
* пустых элементов;
* отображения служебных блоков без контента.

### Причина

Это является техническим дефектом пользовательского интерфейса.
