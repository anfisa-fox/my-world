# Author Mode MVP

Version: 1.0

Status: Approved for Implementation

---

# Цель

Реализовать первую рабочую версию Author Mode для проверки главной продуктовой гипотезы проекта.

Главная гипотеза:

> Автору должно хотеться возвращаться в свой мир и самостоятельно публиковать новый контент непосредственно внутри него.

---

# MVP Scope

В рамках данного этапа реализуются только функции, необходимые для проверки этой гипотезы.

## Hidden Door

* Hidden Door располагается в футере главной страницы.
* Используется символ ✦.
* Текст:

> Этот мир растёт вместе со своим автором ✦

---

## World Recognition

После нажатия ✦:

* открывается небольшое модальное окно;
* мир спрашивает:

> Это ты?

* автор вводит Studio Secret;
* после успешного узнавания отображается короткое сообщение:

> Я узнал тебя.

После этого Author Mode становится активным.

Пользователь остаётся на текущей странице.

---

## Author Mode

После узнавания хозяйки:

* мир не меняет свою структуру;
* отсутствуют Dashboard и административная панель;
* становятся доступны только дополнительные возможности управления контентом.

---

## Wall

Необходимо реализовать полный сценарий создания новой записи.

Автор должен иметь возможность:

* создать новую историю;
* опубликовать её;
* дождаться автоматической публикации.

---

## Author Language

Все сообщения интерфейса должны соответствовать документу:

Author Language.

Система разговаривает спокойно, дружелюбно и по-человечески.

---

# Не входит в MVP

Сознательно исключается из текущего этапа:

* редактирование существующих записей;
* удаление записей;
* создание Artwork;
* создание Characters;
* загрузка изображений;
* World Showcase;
* предпросмотр;
* черновики;
* управление галереей;
* управление персонажами.

Эти возможности уже спроектированы и будут реализовываться после анализа результатов первой демонстрации.

---

# Definition of Done

Работа считается завершённой, если выполнены следующие условия:

* Hidden Door работает на опубликованном сайте.
* Автор может успешно пройти World Recognition.
* Author Mode активируется без перехода на отдельную страницу.
* Автор самостоятельно создаёт новую запись на Wall.
* Запись автоматически публикуется через GitHub и Cloudflare.
* Новая запись появляется на опубликованном сайте без участия разработчика.

---

# Definition of Success

Главным критерием успешности является не техническая реализация.

MVP считается успешным, если после первой демонстрации автор самостоятельно возвращается в свой мир и снова использует Author Mode для публикации новой истории.

Именно желание возвращаться является основной проверяемой гипотезой данного этапа.

---

# Design Freeze

Все продуктовые и UX-решения, описанные в документах:

* Author Experience
* Author Language
* Author Mode Design System
* Author Mode Interaction Flows
* Author Mode MVP

считаются утверждёнными.

До завершения первой пользовательской демонстрации допускаются только технические изменения, необходимые для реализации утверждённой концепции.

Любые изменения пользовательского опыта принимаются только после анализа реального опыта автора.
